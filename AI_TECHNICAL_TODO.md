## القرارات المعتمدة (من النقاشات السابقة)

### الهيكل:
- 5 مساعدين كهدف نهائي (نبدأ بـ 2 فقط: طبيب + تيم ليدر)
- بدون Orchestrator (مساعد واحد لكل مستخدم حسب دوره)
- نواة مشتركة + برومبتات معزولة لكل مساعد

### الصلاحيات:
- مصدر واحد للحقيقة: `permissions.ts`
- الذكاء = صلاحيات المستخدم بالضبط (لا أقل، لا أكثر)
- DCM AI يقدر يقترح تشخيصات طبية (لكن الطبيب يقرر، لا وصفات)

### الاسم:
- DCM AI

### اللغة:
- البرومبتات بالإنجليزي (دقة + توفير توكنات)
- الذكاء يرد بلغة المستخدم

### الكاش (Prompt Caching):
- **TTL: ساعة كاملة** (extended cache) - مطبّق بـ `lib/ai.ts`
- يتجدد مع كل استخدام
- مشترك بين كل المراكز (نفس الـ API Key)
- توفير ~90% بعد الاستدعاء الأول

### استراتيجية RAG:
- **النواة:** معرفة عامة فقط (تعاريف، مفاهيم عامة)
- **كل مساعد:** RAG منفصل خاص فيه يحتوي قوانينه وقدراته
- **السبب:** عزل أمني تام (تطبيق "Data Visibility = UI Visibility")
- نقدر نسترجع 3-5 عناوين متعلقة لكل طلب (تكلفة ~$0.002 إضافية)
- حجم قاعدة المعرفة لا يأثر على التكلفة لكل طلب
- التخزين: Supabase pgvector

### النواة (Core Prompt) - مكتملة:
- ملف: `lib/ai/core/corePrompt.ts` (DRAFT)
- 8 أقسام: Identity, Core Values, Golden Rule, Core Behaviors, Boundaries, App Context, Permissions Awareness, Communication Style
- ~2,050 توكن
- مدمجة 6 محافظات صحية بالكويت
- مدمجة الشفتات (صباحي 7-2، مسائي 2-9)
- مدمجة الفترات P1-P4 بأوقاتها
